All requests go to the base URL: `https://exchange.jamesxu.dev`

A machine-readable OpenAPI schema and Swagger UI are also available:

```text
https://exchange.jamesxu.dev/api-doc/openapi.json
https://exchange.jamesxu.dev/docs
```

## Public endpoints

No authentication required.

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/health` | Service and persistence health |
| `GET` | `/api/v1/markets` | List configured markets |

<AccordionGroup>
  <Accordion title="GET /health">
    Returns the service status and current persistence state. No authentication required.

    **Response fields**

    <ResponseField name="status" type="string" required>
      `ok` when persistence is `disabled` or `ok`. `degraded` when persistence is `retrying`, `backpressured`, or `stopped`.
    </ResponseField>
    <ResponseField name="service" type="string" required>
      Always `exchange`.
    </ResponseField>
    <ResponseField name="now" type="string" required>
      Current server time in RFC 3339 format.
    </ResponseField>
    <ResponseField name="persistence" type="object" required>
      Current persistence status including mode.
    </ResponseField>

    ```bash
    curl https://exchange.jamesxu.dev/health
    ```

    ```json
    {
      "status": "ok",
      "service": "exchange",
      "now": "2026-03-18T15:27:00.000000Z",
      "persistence": {
        "backend": "in_memory",
        "mode": "disabled",
        "queue_depth": 0,
        "last_error": null
      }
    }
    ```
  </Accordion>

        Returns all open positions for the authenticated trader.

        ```bash
        curl \
          -H "x-api-key: YOUR_API_KEY" \
          https://exchange.jamesxu.dev/api/v1/positions
        ```

        ```json
        [
          {
            "market": "BTC-USD",
        ```

      <Accordion title="POST /api/v1/orders">
        Submits a limit order. Returns the order, any fills that occurred, and whether the order is resting in the book.

        **Request body**

        <ParamField path="body.market" type="string" required>
          Market symbol, e.g. `BTC-USD`. Must follow the `BASE-QUOTE` format.
        </ParamField>
        <ParamField path="body.side" type="string" required>
          `BUY` or `SELL`.
        </ParamField>
        <ParamField path="body.price" type="number" required>
          Limit price. Must be greater than zero and align to the market's `tick_size`.
        </ParamField>
        <ParamField path="body.quantity" type="number" required>
          Order quantity. Must be at least the market's `min_order_quantity`.
        </ParamField>

        **Response fields**

        <ResponseField name="order" type="object" required>
          The submitted order with its assigned `id` and current `remaining` quantity.
        </ResponseField>
        <ResponseField name="fills" type="array" required>
          List of fills that occurred immediately on submission. Empty if the order rested.
        </ResponseField>
        <ResponseField name="resting" type="boolean" required>
          `true` if the order has remaining quantity in the book after matching.
        </ResponseField>

        ```bash
        curl \
          -X POST \
          -H "x-api-key: YOUR_API_KEY" \
          -H "content-type: application/json" \
          https://exchange.jamesxu.dev/api/v1/orders \
          -d '{"market":"BTC-USD","side":"BUY","price":100,"quantity":2}'
        ```

        ```json
        {
          "order": {
            "id": "66377526-7b98-485a-8c40-7024e68fa3c5",
            "trader_id": "9d64e278-8b56-46e9-9cab-18c5b22f2fb9",
            "market": "BTC-USD",
            "side": "BUY",
            "price": 100,
            "quantity": 2,
            "remaining": 2,
            "created_at": "2026-03-18T15:28:01.342510Z"
          },
          "fills": [],
          "resting": true
        }
        ```

        <Warning>
          Order admission checks worst-case net exposure from all resting open orders. A position limit of **+/-1000** applies per market. Exceeding this returns `409 Conflict`.
        </Warning>
      </Accordion>

      <Accordion title="PATCH /api/v1/orders/{order_id}">
        Amends the remaining quantity of a resting order. You can only reduce `remaining` — increasing it is not allowed.

        **Path parameters**

        <ParamField path="path.order_id" type="string" required>
          UUID of the resting order to amend.
        </ParamField>

        **Request body**

        <ParamField path="body.remaining" type="number" required>
          New remaining quantity. Must be greater than zero and less than or equal to the current remaining.
        </ParamField>

        ```bash
        curl \
          -X PATCH \
          -H "x-api-key: YOUR_API_KEY" \
          -H "content-type: application/json" \
          https://exchange.jamesxu.dev/api/v1/orders/ORDER_ID \
          -d '{"remaining":1}'
        ```

        ```json
        {
          "order": {
            "id": "66377526-7b98-485a-8c40-7024e68fa3c5",
            "trader_id": "9d64e278-8b56-46e9-9cab-18c5b22f2fb9",
            "market": "BTC-USD",
            "side": "BUY",
            "price": 100,
            "quantity": 2,
            "remaining": 1,
            "created_at": "2026-03-18T15:28:01.342510Z"
          }
        }
        ```
      </Accordion>

      <Accordion title="POST /api/v1/admin/markets">
        Creates or replaces a market definition. If a market with the same `market_id` already exists, it is replaced.

        **Request body**

        <ParamField path="body.market_id" type="string" required>
          Market symbol in `BASE-QUOTE` format, e.g. `BTC-USD`.
        </ParamField>
        <ParamField path="body.display_name" type="string">
          Optional. Human-readable market name. Falls back to `market_id` if omitted.
        </ParamField>
        <ParamField path="body.base_asset" type="string" required>
          Base asset symbol.
        </ParamField>
        <ParamField path="body.quote_asset" type="string" required>
          Quote asset symbol.
        </ParamField>
        <ParamField path="body.tick_size" type="number" required>
          Minimum price increment. All order prices must be a multiple of this value.
        </ParamField>
        <ParamField path="body.min_order_quantity" type="number" required>
          Minimum order quantity. Orders below this are rejected.
        </ParamField>
        <ParamField path="body.reference_price" type="number">
          Optional reference price used for marking when no trades have occurred.
        </ParamField>
        <ParamField path="body.enabled" type="boolean" required>
          Whether the market is open for trading on creation. `true` creates an enabled market, `false` creates a disabled one.
        </ParamField>

        ```bash
        curl \
          -X POST \
          -H "Authorization: Bearer ADMIN_API_TOKEN" \
          -H "content-type: application/json" \
          https://exchange.jamesxu.dev/api/v1/admin/markets \
          -d '{
            "market_id":"BTC-USD",
            "display_name":"Bitcoin",
            "base_asset":"BTC",
            "quote_asset":"USD",
            "tick_size":1,
            "min_order_quantity":1,
            "reference_price":100,
            "enabled":true
          }'
        ```

      <Accordion title="POST /api/v1/admin/markets/{market_id}/settle">
        Settles a market. Cancels all resting orders, then closes every trader's net position at the supplied settlement price, realizing PnL. Positions are flattened to zero after settlement.

        **Path parameters**

        <ParamField path="path.market_id" type="string" required>
          Market symbol to settle.
        </ParamField>

        **Request body**

        <ParamField path="body.settlement_price" type="number" required>
          The admin-determined true value per share. PnL is realized against each trader's signed net position at this price. Must be greater than zero.
        </ParamField>
        <ParamField path="body.announcement" type="string">
          Optional. Custom announcement text broadcast to all WebSocket clients after settlement. Defaults to a generated message if omitted.
        </ParamField>

        ```bash
        curl \
          -X POST \
          -H "Authorization: Bearer ADMIN_API_TOKEN" \
          -H "content-type: application/json" \
          https://exchange.jamesxu.dev/api/v1/admin/markets/BTC-USD/settle \
          -d '{"settlement_price":150}'
        ```

        ```json
        {
          "market": {
            "market_id": "BTC-USD",
            "status": "settled",
            "settlement_price": 150
          },
          "canceled_orders": 5,
          "affected_traders": 3,
          "settled_quantity": 200,
          "settlement_price": 150
        }
        ```
      </Accordion>

      <Accordion title="POST /api/v1/admin/users/reset">
        Clears all user positions, open orders, and fills exchange-wide. This is a destructive operation intended for resetting between competition rounds.

        <Warning>
          This operation is irreversible. All trader positions, open orders, and fills are permanently deleted.
        </Warning>

        ```bash
        curl \
          -X POST \
          -H "Authorization: Bearer ADMIN_API_TOKEN" \
          https://exchange.jamesxu.dev/api/v1/admin/users/reset
        ```

## Data model notes

- `side` is `BUY` or `SELL`
- `price`, `quantity`, and `remaining` are unsigned 64-bit integers (`u64`)
- The per-market net position limit is **+/-1000**
- Traders can sell from flat and go short; order admission is based on worst-case net exposure if all resting open orders were to execute
- Realized PnL is tracked on positions; no pre-seeded inventory or cash balance is required to open a long or short position
- Leaderboard ranking uses settled prices, orderbook marks, or market reference prices
- Pagination is not yet implemented
